# Sprint 3 brief: 選定記事から投稿本文を生成できる

## 対象プラットフォーム
`web`(画面を持たないバックエンド自動化)。

## 含まれる機能(仕様書「機能一覧」より原文)

### F3. ポスト文面の生成
- 説明: 選定した記事をもとに、日本語で読みやすく、そのニュースの要点が伝わる投稿文面を生成する。煽りすぎず正確で、AIニュースアカウントとして自然なトーン。
- 優先度: Must
- 受け入れ基準:
  - [ ] 実行すると、選定記事に対応した日本語の投稿本文が生成される。
  - [ ] 本文は記事の要点(何が・どう新しいか)を含み、記事タイトルの丸写しではなく要約・言い換えになっている。
  - [ ] 事実に基づき、記事に書かれていない断定的な誇張や虚偽情報を含まない(検証: 生成文の主張が元記事の内容の範囲に収まっている)。
  - [ ] トーン・言語(日本語)は設定で一貫しており、毎回極端にブレない。
  - [ ] 文面生成に失敗した場合(APIエラー等)は投稿せず、エラーを記録して安全に終了する(壊れた/空の投稿をしない)。

## このスプリントの受け入れ基準(デモできる状態・仕様書「スプリント計画」より原文)
- [ ] 選定記事に対応する日本語の投稿本文が生成される。
- [ ] 本文がタイトルの丸写しでなく要約・言い換えになっており、記事の要点を含む。
- [ ] 生成文の主張が元記事の内容の範囲に収まり、明らかな誇張・虚偽を含まない(サンプル記事で目視確認できる)。
- [ ] 文面生成APIが失敗したケースを擬似的に起こしても、投稿処理に進まず安全に終了しエラーが記録される。

## 評価基準(仕様書より原文)
- 致命的バグ: 0件 / コンソールエラー: 0件 / 受け入れ基準充足率: 100%

## 関連するリスク・留意事項(仕様書より原文抜粋・外部サービス依存)
- **Anthropic API(Claude)のAPIキー発行**: ニュース要約・文面生成に Claude を用いる場合、利用者本人が Anthropic のアカウントで API キーを発行し、従量課金(有料)を負担する必要がある。キー未設定では文面生成が行えない。
- 認証情報(APIキー等)はソースコードや公開リポジトリに平文で含めてはならない。`.env`等の設定値経由でのみ読み込み、リポジトリには実値を残さず、キー名のみの雛形(`.env.example`)を残す。

## 事前確認の結果(オーケストレーターがユーザーに確認済み)
Anthropic APIキーは利用者が「今すぐ用意する」と回答済み。ただしこのスプリント実装時点でまだキーが`.env`に設定されていない場合は、キー未設定時にエラーを検知して安全に終了する実装(上記受け入れ基準通り)を優先し、キーが用意でき次第、実際の生成呼び出しで動作確認する。
