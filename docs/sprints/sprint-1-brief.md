# Sprint 1 brief: 急上昇AIニュースを収集・スコアリングできる

## 対象プラットフォーム
`web`(画面を持たないバックエンド自動化)。UIスクリーンを持たず、外部トリガーによって定時起動され投稿処理を行うサーバーサイド/バッチ型の自動化システムとして構築する。

## 含まれる機能(仕様書「機能一覧」より原文)

### F1. 急上昇AIニュースの収集とスコアリング
- 説明: Web上のAIニュース情報源から直近の記事候補を収集し、「今まさに注目度が急上昇している」度合いをスコア化して候補リストを作る。単なる新着ではなく、直近で話題性が急伸しているものを上位に並べる。
- 優先度: Must
- 受け入れ基準:
  - [ ] 実行すると、少なくとも10件以上のAIニュース候補が「タイトル・元記事URL・情報源・推定公開時刻・急上昇スコア」を持つ構造化データとして得られる。
  - [ ] 各候補に急上昇スコアが付与され、候補リストがスコア降順に並ぶ。
  - [ ] スコアリングは「新しさ(直近性)」と「話題の伸び(複数ソースでの言及・急な言及増加等の話題性シグナル)」の両方を反映し、公開が古く話題も伸びていない記事は上位に来ない(検証: 意図的に古い低話題の記事を混ぜても上位3件に入らない)。
  - [ ] 収集対象がAI関連(生成AI・LLM・機械学習・AI製品/企業/研究等)に限定され、無関係ジャンルの記事は候補に含まれない。
  - [ ] 情報源への通信失敗時も処理全体は落ちず、取得できたソース分だけで候補リストを構築し、失敗したソースはログに記録される。

## このスプリントの受け入れ基準(デモできる状態・仕様書「スプリント計画」より原文)
- [ ] コマンド1つで実行すると、10件以上のAIニュース候補が「タイトル・URL・情報源・推定公開時刻・急上昇スコア」を持ってスコア降順で出力される。
- [ ] 意図的に「古く話題も伸びていない記事」を混ぜても上位3件に入らないことを確認できる。
- [ ] AI無関係の記事が候補に混ざらない。
- [ ] 一部の情報源が通信失敗しても処理は落ちず、取得できた分で候補を作りエラーはログに残る。

## 評価基準(仕様書より原文)
- 致命的バグ: 0件
- コンソールエラー: 0件(想定内のソース通信失敗をハンドリングしたログ出力は「エラー」に数えない)
- 上記受け入れ基準の充足率: 100%
