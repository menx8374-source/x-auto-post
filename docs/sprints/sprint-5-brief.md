# Sprint 5 brief: ドライランで投稿予定の全ツイートをプレビューできる

## 対象プラットフォーム
`web`(画面を持たないバックエンド自動化)。

## 含まれる機能(仕様書「機能一覧」より原文)
含まれる機能: F11(+ Sprint1〜4を統合した「投稿予定生成」パイプライン)。このスプリントでSprint1〜4で作った収集→選定→生成→分割→リンク付与の各処理を1本のパイプラインとして統合する。

### F11. ドライラン(投稿せずプレビュー)
- 説明: 実際にXへ投稿せずに、「その回に何を投稿する予定か」(選定記事・生成本文・分割後の各ツイート・リンクツイート)を確認できるモード。テスト・レート制限節約・規約リスク回避に使う。
- 優先度: Must
- 受け入れ基準:
  - [ ] ドライランを指定して実行すると、Xへは1件も投稿されず、投稿予定の全ツイート(順序・各文字数・リンクツイート含む)が出力される。
  - [ ] ドライランでも記事収集・選定・文面生成・文字数計算・分割まで本番と同じ処理を通り、本番投稿時との差異が「実際に送信するか否か」だけである。
  - [ ] ドライランでは投稿履歴(既出判定用)を汚さない、または汚さないよう選べる。

## このスプリントの受け入れ基準(デモできる状態・仕様書「スプリント計画」より原文)
- [ ] ドライラン指定で実行すると、Xへ1件も投稿せず、収集→選定→生成→分割→リンク付与までを通し、投稿予定の全ツイート(順序・各文字数・リンクツイート)を出力する。
- [ ] ドライランが投稿履歴(既出判定用)を汚さない、または汚さない選択ができる。
- [ ] 本番投稿処理との差異が「実際に送信するか否か」だけであることがコード上/出力上で確認できる。

## 評価基準(仕様書より原文)
- 致命的バグ: 0件 / コンソールエラー: 0件 / 受け入れ基準充足率: 100%
